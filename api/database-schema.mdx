## Overview

TextileFlow ERP uses **Firebase Firestore** as its NoSQL database. The schema is designed for scalability, real-time updates, and efficient querying. This page documents all collections, document structures, and relationships.

---

## Collections Overview

| Collection | Purpose | Key Relationships |
|------------|---------|------------------|
| `usuario` | User accounts and profiles | Referenced by attendances, payments |
| `attendances` | Check-in/out records | Links to usuario, schedules |
| `attendance_summary` | Daily attendance summaries | Aggregates from attendances |
| `schedules` | Work schedule configuration | Used by attendances for validation |
| `articulo` | Inventory items | Links to categoria |
| `categoria` | Product categories | Referenced by articulo |
| `atributo` | Product attributes (Color, Size) | Used by articulo |
| `valorAtributo` | Attribute values (Red, Large) | Links to atributo |
| `pago` | Employee payment records | Links to usuario |

## Attendance Collections

### `attendances` Collection

Stores individual check-in and check-out records.

**Document ID:** Auto-generated by Firestore

<ParamField path="userId" type="string" required>
  Reference to usuario.uid
</ParamField>

<ParamField path="userEmail" type="string" required>
  Employee email (denormalized for quick access)
</ParamField>

<ParamField path="displayName" type="string" required>
  Employee name (denormalized)
</ParamField>

<ParamField path="role" type="string">
  Employee role (denormalized)
</ParamField>

<ParamField path="date" type="string" required>
  Date in YYYY-MM-DD format
</ParamField>

<ParamField path="timestamp" type="Timestamp" required>
  Firestore timestamp of the record
</ParamField>

<ParamField path="type" type="string" required>
  "entrada" (check-in) or "salida" (check-out)
</ParamField>

<ParamField path="actualTime" type="string" required>
  Actual time in HH:MM format
</ParamField>

<ParamField path="scheduledTime" type="string">
  Expected time from schedule (HH:MM)
</ParamField>

<ParamField path="status" type="string">
  "on-time", "late", or "early"
</ParamField>

<ParamField path="minutesDifference" type="number">
  Minutes difference from scheduled time (+ = late, - = early)
</ParamField>

<ParamField path="source" type="string" required>
  "self" (employee) or "manual" (admin)
</ParamField>

<ParamField path="registeredBy" type="string">
  Admin user ID (for manual registrations)
</ParamField>

<ParamField path="notes" type="string">
  Optional notes or comments
</ParamField>

<ParamField path="editedBy" type="string">
  Admin ID who last edited (if updated)
</ParamField>

<ParamField path="editHistory" type="array">
  Array of edit records with timestamps and changes
</ParamField>

<ParamField path="createdAt" type="Timestamp">
  Server timestamp of creation
</ParamField>

<ParamField path="updatedAt" type="Timestamp">
  Server timestamp of last update
</ParamField>

**Indexes:**
- Composite: `userId` (asc) + `date` (asc)
- Composite: `date` (asc) + `type` (asc)
- Composite: `userId` (asc) + `date` (asc) + `timestamp` (desc)
- Composite: `date` (asc) + `timestamp` (desc)

**Queries:**
```javascript
// Get all attendances for today
where('date', '==', today).orderBy('timestamp', 'desc')

// Get user's attendances for date range
where('userId', '==', userId)
  .where('date', '>=', startDate)
  .where('date', '<=', endDate)
  .orderBy('date', 'desc')
```

### `attendance_summary` Collection

Daily aggregated attendance data for quick reporting.

**Document ID:** `{userId}_{date}` (e.g., "user123_2024-03-15")

<ParamField path="userId" type="string" required>
  Reference to usuario.uid
</ParamField>

<ParamField path="date" type="string" required>
  Date in YYYY-MM-DD format
</ParamField>

<ParamField path="displayName" type="string" required>
  Employee name (denormalized)
</ParamField>

<ParamField path="checkIn" type="Timestamp">
  Check-in timestamp (null if absent)
</ParamField>

<ParamField path="checkOut" type="Timestamp">
  Check-out timestamp (null if not checked out)
</ParamField>

<ParamField path="hoursWorked" type="number">
  Total hours worked (calculated from checkIn and checkOut)
</ParamField>

<ParamField path="status" type="string" required>
  "complete", "incomplete", or "absent"
</ParamField>

<ParamField path="wasLate" type="boolean">
  True if employee was late for check-in
</ParamField>

<ParamField path="leftEarly" type="boolean">
  True if employee left before scheduled time
</ParamField>

<ParamField path="lastUpdated" type="Timestamp">
  Server timestamp of last update
</ParamField>

<CodeGroup>
```javascript Example Complete Day
{
  // Document ID: "abc123xyz_2024-03-15"
  userId: "abc123xyz",
  date: "2024-03-15",
  displayName: "John Doe",
  checkIn: Timestamp(2024-03-15 08:05:00),
  checkOut: Timestamp(2024-03-15 17:00:00),
  hoursWorked: 8.92,
  status: "complete",
  wasLate: false,
  leftEarly: false,
  lastUpdated: Timestamp
}
```

```javascript Example Incomplete Day
{
  // Document ID: "abc123xyz_2024-03-16"
  userId: "abc123xyz",
  date: "2024-03-16",
  displayName: "John Doe",
  checkIn: Timestamp(2024-03-16 08:30:00),
  checkOut: null,
  hoursWorked: null,
  status: "incomplete",
  wasLate: true,
  leftEarly: false,
  lastUpdated: Timestamp
}
```

```javascript Example Absent Day
{
  // Document ID: "abc123xyz_2024-03-17"
  userId: "abc123xyz",
  date: "2024-03-17",
  displayName: "John Doe",
  checkIn: null,
  checkOut: null,
  hoursWorked: null,
  status: "absent",
  wasLate: false,
  leftEarly: false,
  lastUpdated: Timestamp
}
```
</CodeGroup>

**Indexes:**
- Composite: `date` (asc)
- Composite: `userId` (asc) + `date` (asc)
- Composite: `status` (asc) + `date` (asc)

### `schedules` Collection

Work schedule configuration for attendance validation.

**Document ID:** Auto-generated by Firestore

<ParamField path="date" type="string">
  Specific date (YYYY-MM-DD) - omitted for default schedule
</ParamField>

<ParamField path="entryTime" type="string" required>
  Expected check-in time (HH:MM format)
</ParamField>

<ParamField path="exitTime" type="string" required>
  Expected check-out time (HH:MM format)
</ParamField>

<ParamField path="toleranceMinutes" type="number" required>
  Allowed minutes before/after scheduled time
</ParamField>

<ParamField path="workHours" type="number" required>
  Expected work hours (auto-calculated)
</ParamField>

<ParamField path="isDefault" type="boolean" required>
  True if this is the default schedule
</ParamField>

<ParamField path="appliesTo" type="string" required>
  "all" or specific user IDs
</ParamField>

<ParamField path="setBy" type="string">
  Admin user ID who created this schedule
</ParamField>

<ParamField path="createdAt" type="Timestamp">
  Creation timestamp
</ParamField>

<ParamField path="updatedAt" type="Timestamp">
  Last update timestamp
</ParamField>

## Relationships Diagram

```
usuario (uid)
  ↓
  ├─→ attendances (userId)
  ├─→ attendance_summary (userId)
  └─→ pago (userId)

schedules
  ↓
  └─→ attendances (validates against)

categoria (id)
  ↓
  └─→ articulo (categoriaId)

atributo (id)
  ↓
  ├─→ valorAtributo (atributoId)
  └─→ articulo.atributos (key)
```
