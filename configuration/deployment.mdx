## Overview

TextileFlow ERP is designed to be deployed on Firebase Hosting, providing a fast, secure, and scalable hosting solution for your application. This guide covers the complete deployment process from local development to production.

## Prerequisites

Before deploying, ensure you have:

- ✅ Node.js v14 or higher installed
- ✅ Firebase CLI installed globally
- ✅ Firebase project created and configured
- ✅ Admin access to your Firebase project
- ✅ All configuration files properly set up

## Firebase Hosting Configuration

TextileFlow uses the following hosting configuration in `firebase.json`:

<CodeGroup>
```json firebase.json
{
  "hosting": {
    "public": "public",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ]
  },
  "firestore": {
    "database": "(default)",
    "location": "nam5",
    "rules": "firestore.rules",
    "indexes": "firestore.indexes.json"
  },
  "storage": {
    "rules": "storage.rules"
  }
}
```
</CodeGroup>

### Key Configuration Points

- **public**: The `public` directory contains all files to be deployed
- **ignore**: Excludes configuration files and node_modules from deployment
- **rules**: Points to Firestore security rules file
- **indexes**: Points to Firestore index configuration

## Pre-Deployment Checklist

<Steps>
  <Step title="Update Firebase Configuration">
    Ensure `public/src/config/firebase.js` has your production Firebase credentials:
    
    <CodeGroup>
    ```javascript firebase.js
    const firebaseConfig = {
      apiKey: "YOUR_PRODUCTION_API_KEY",
      authDomain: "your-project.firebaseapp.com",
      projectId: "your-production-project-id",
      storageBucket: "your-project.firebasestorage.app",
      messagingSenderId: "YOUR_SENDER_ID",
      appId: "YOUR_APP_ID",
      measurementId: "YOUR_MEASUREMENT_ID"
    };
    ```
    </CodeGroup>
  </Step>

  <Step title="Update Firestore Security Rules">
    Replace development rules with production rules in `firestore.rules`:
    
    <CodeGroup>
    ```javascript firestore.rules
    rules_version = '2';
    
    service cloud.firestore {
      match /databases/{database}/documents {
        // Require authentication for all reads/writes
        match /{document=**} {
          allow read, write: if request.auth != null;
        }
        
        // User-specific rules
        match /usuario/{userId} {
          allow read: if request.auth != null;
          allow write: if request.auth.uid == userId ||
                         get(/databases/$(database)/documents/usuario/$(request.auth.uid)).data.rol == 'admin';
        }
        
        // Admin-only collections
        match /configuracion_sistema/{document} {
          allow read: if request.auth != null;
          allow write: if get(/databases/$(database)/documents/usuario/$(request.auth.uid)).data.rol == 'admin';
        }
      }
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Remove Debug Code">
    - Remove console.log statements (or use a logger with levels)
    - Disable emulator connections in production
    - Remove development-only features
  </Step>

  <Step title="Test Locally">
    Run the application locally with production configuration:
    
    ```bash
    firebase serve
    ```
    
    Visit `http://localhost:5000` and test all features
  </Step>

  <Step title="Review Environment Detection">
    Ensure emulators only connect in development:
    
    <CodeGroup>
    ```javascript Conditional Emulator Connection
    if (location.hostname === 'localhost' || location.hostname === '127.0.0.1') {
      // Development - use emulators
      connectAuthEmulator(auth, 'http://localhost:9099');
      connectFirestoreEmulator(db, 'localhost', 8080);
    } else {
      // Production - use Firebase Cloud
      console.log('✓ Usando Firebase Cloud');
    }
    ```
    </CodeGroup>
  </Step>
</Steps>

<Warning>
  Never deploy with development Firestore rules (`allow read, write: if true`). This exposes your database to unauthorized access.
</Warning>

## Installation and Setup

### Install Firebase CLI

If you haven't installed Firebase CLI yet:

```bash
npm install -g firebase-tools
```

Verify installation:

```bash
firebase --version
```

### Login to Firebase

```bash
firebase login
```

This will open a browser window for authentication.

### Initialize Firebase Project

If starting fresh:

```bash
cd /path/to/textileflow
firebase init
```

Select:
- ✅ Firestore
- ✅ Hosting
- ✅ Storage (if using file uploads)

## Deployment Process

### Deploy Everything

Deploy hosting, Firestore rules, and storage rules:

<CodeGroup>
```bash Full Deployment
firebase deploy
```
</CodeGroup>

This command will:
1. Upload all files from the `public` directory
2. Deploy Firestore security rules
3. Deploy Storage security rules
4. Generate a deployment URL

### Deploy Only Hosting

If you only changed frontend code:

<CodeGroup>
```bash Hosting Only
firebase deploy --only hosting
```
</CodeGroup>

### Deploy Only Rules

If you only changed security rules:

<CodeGroup>
```bash Rules Only
firebase deploy --only firestore:rules
firebase deploy --only storage:rules
```
</CodeGroup>

### Deploy to Specific Project

If you have multiple Firebase projects:

<CodeGroup>
```bash Specific Project
firebase deploy --project your-project-id
```
</CodeGroup>

## Post-Deployment Steps

<Steps>
  <Step title="Verify Deployment">
    1. Visit your Firebase Hosting URL (shown after deployment)
    2. Test the login functionality
    3. Navigate through all modules
    4. Test creating and editing data
    5. Verify all API calls work correctly
  </Step>

  <Step title="Configure Custom Domain (Optional)">
    1. Go to Firebase Console → Hosting
    2. Click **Add custom domain**
    3. Follow the DNS configuration instructions
    4. Wait for SSL certificate provisioning (can take up to 24 hours)
  </Step>

  <Step title="Create Initial Admin User">
    Use Firebase Console to create the first admin user:
    
    1. Go to **Authentication** → **Users**
    2. Click **Add user**
    3. Enter email and password
    4. Go to **Firestore Database**
    5. Create a document in `usuario` collection:
    
    <CodeGroup>
    ```json Initial Admin User
    {
      "uid": "<firebase-auth-uid>",
      "email": "admin@yourcompany.com",
      "nombre": "Admin",
      "apellido": "User",
      "rol": "admin",
      "estado": "activo",
      "fechaCreacion": "2026-03-08T00:00:00Z"
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure Environment Variables (If Needed)">
    For sensitive configuration, use environment variables:
    
    1. Create `.env` file (don't commit to git)
    2. Use a build process to inject variables
    3. Or use Firebase Remote Config
  </Step>

  <Step title="Set Up Monitoring">
    Enable monitoring in Firebase Console:
    
    - **Performance Monitoring**: Track page load times
    - **Crashlytics**: Monitor application errors
    - **Analytics**: Track user behavior
  </Step>
</Steps>

## Continuous Deployment

### GitHub Actions

Automate deployments with GitHub Actions:

<CodeGroup>
```yaml .github/workflows/firebase-hosting.yml
name: Deploy to Firebase Hosting

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v2
      
      - name: Deploy to Firebase
        uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: '${{ secrets.GITHUB_TOKEN }}'
          firebaseServiceAccount: '${{ secrets.FIREBASE_SERVICE_ACCOUNT }}'
          channelId: live
          projectId: your-project-id
```
</CodeGroup>

### Setup GitHub Actions

<Steps>
  <Step title="Generate Service Account">
    1. Go to Firebase Console → Project Settings
    2. Navigate to **Service Accounts**
    3. Click **Generate new private key**
    4. Save the JSON file securely
  </Step>

  <Step title="Add Secret to GitHub">
    1. Go to GitHub repository → Settings → Secrets
    2. Click **New repository secret**
    3. Name: `FIREBASE_SERVICE_ACCOUNT`
    4. Value: Paste the entire JSON content
  </Step>

  <Step title="Push and Deploy">
    Push to main branch to trigger automatic deployment
  </Step>
</Steps>

## Performance Optimization

<Steps>
  <Step title="Enable Caching">
    Add cache headers in `firebase.json`:
    
    <CodeGroup>
    ```json Caching Configuration
    {
      "hosting": {
        "public": "public",
        "headers": [
          {
            "source": "**/*.@(jpg|jpeg|gif|png|svg|webp)",
            "headers": [
              {
                "key": "Cache-Control",
                "value": "max-age=31536000"
              }
            ]
          },
          {
            "source": "**/*.@(js|css)",
            "headers": [
              {
                "key": "Cache-Control",
                "value": "max-age=31536000"
              }
            ]
          }
        ]
      }
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Optimize Assets">
    - Minify JavaScript and CSS files
    - Compress images (use WebP format)
    - Use lazy loading for images
    - Bundle and optimize JavaScript modules
  </Step>

  <Step title="Configure CDN">
    Firebase Hosting automatically uses a global CDN, but you can optimize:
    
    - Ensure proper cache headers
    - Use Firebase Hosting CDN for static assets
    - Consider Firebase Cloud Storage for large files
  </Step>
</Steps>

## Security Best Practices

<Steps>
  <Step title="Enable HTTPS">
    Firebase Hosting automatically provides SSL certificates. Ensure:
    
    - All requests are served over HTTPS
    - HTTP automatically redirects to HTTPS
    - No mixed content warnings
  </Step>

  <Step title="Configure CORS">
    If using external APIs, configure CORS properly
  </Step>

  <Step title="Review Security Rules Regularly">
    - Audit Firestore rules monthly
    - Test rules with Firebase Rules Playground
    - Monitor for unauthorized access attempts
  </Step>

  <Step title="Enable App Check">
    Protect your backend resources:
    
    1. Go to Firebase Console → App Check
    2. Register your app
    3. Enable enforcement for Firestore and Storage
  </Step>
</Steps>

## Troubleshooting Deployment Issues

### Issue: Deployment Fails

**Solution:**
1. Check Firebase CLI version: `firebase --version`
2. Ensure you're logged in: `firebase login --reauth`
3. Verify project selection: `firebase use --add`
4. Check for syntax errors in configuration files

### Issue: Site Shows Old Version

**Solution:**
1. Clear browser cache
2. Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
3. Check Firebase Hosting dashboard for deployment status
4. Verify cache headers aren't too aggressive

### Issue: 404 Errors After Deployment

**Solution:**
1. Verify `public` directory structure
2. Check that all referenced files exist
3. Review `firebase.json` hosting configuration
4. Ensure file paths are correct (case-sensitive on some systems)

### Issue: Authentication Not Working

**Solution:**
1. Verify authorized domains in Firebase Console → Authentication → Settings
2. Add your custom domain to authorized domains
3. Check that Firebase config is correct for production

```bash Rollback to Previous Version
# Rollback using Firebase Console
# Go to Hosting → Release history
# Click the three dots next to a previous release
# Select "Restore"
```
</CodeGroup>

## Cost Management

Monitor and optimize Firebase usage:

- **Firestore**: 
  - Optimize queries to reduce reads
  - Use Firestore Rules to prevent abuse
  - Consider document structure to minimize reads

- **Storage**:
  - Compress images before upload
  - Set lifecycle policies to delete old files
  - Use appropriate storage buckets

- **Hosting**:
  - Firebase Hosting is free up to 10GB/month
  - Monitor bandwidth usage in Firebase Console
