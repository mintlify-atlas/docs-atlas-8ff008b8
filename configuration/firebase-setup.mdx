## Overview

TextileFlow ERP uses Firebase as its backend infrastructure, providing authentication, database, storage, and hosting services. This guide walks you through setting up Firebase for your TextileFlow installation.

## Prerequisites

Before you begin, ensure you have:

- A Google account
- Node.js v14 or higher installed
- Firebase CLI installed globally
- Access to the Firebase Console

## Firebase Project Setup

<Steps>
  <Step title="Create a Firebase Project">
    1. Go to the [Firebase Console](https://console.firebase.google.com/)
    2. Click **Add project** or **Create a project**
    3. Enter your project name (e.g., "textileflow-erp")
    4. Accept the Firebase terms and click **Continue**
    5. Choose whether to enable Google Analytics (recommended)
    6. Click **Create project**
  </Step>

  <Step title="Enable Firebase Services">
    Enable the following services in your Firebase project:

    **Authentication**
    - Navigate to **Build** → **Authentication**
    - Click **Get started**
    - Enable **Email/Password** sign-in method

    **Firestore Database**
    - Navigate to **Build** → **Firestore Database**
    - Click **Create database**
    - Start in **Production mode** or **Test mode** (for development)
    - Select your database location (e.g., `nam5`)

    **Storage**
    - Navigate to **Build** → **Storage**
    - Click **Get started**
    - Accept default security rules

    **Hosting**
    - Navigate to **Build** → **Hosting**
    - Click **Get started**
    - Follow the setup wizard
  </Step>

  <Step title="Get Firebase Configuration">
    1. In Firebase Console, go to **Project Settings** (gear icon)
    2. Scroll down to **Your apps** section
    3. Click the web icon (`</>`)
    4. Register your app with a nickname
    5. Copy the Firebase configuration object
  </Step>
</Steps>

## Firebase Configuration File

Create or update `public/src/config/firebase.js` with your Firebase configuration:

<CodeGroup>
```javascript public/src/config/firebase.js
import { initializeApp } from "https://www.gstatic.com/firebasejs/10.13.1/firebase-app.js";
import { getAuth, connectAuthEmulator } from "https://www.gstatic.com/firebasejs/10.13.1/firebase-auth.js";
import { getFirestore, connectFirestoreEmulator } from "https://www.gstatic.com/firebasejs/10.13.1/firebase-firestore.js";
import { getStorage } from "https://www.gstatic.com/firebasejs/10.13.1/firebase-storage.js";
import { getAnalytics } from "https://www.gstatic.com/firebasejs/10.13.1/firebase-analytics.js";

const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project.firebasestorage.app",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID",
  measurementId: "YOUR_MEASUREMENT_ID"
};

const app = initializeApp(firebaseConfig);

export const auth = getAuth(app);
export const db = getFirestore(app);
export const storage = getStorage(app);
export const analytics = getAnalytics(app);

export default app;
```
</CodeGroup>

<Warning>
  Never commit your Firebase configuration with real credentials to public repositories. Use environment variables for production deployments.
</Warning>

## Firebase Configuration Structure

The `firebase.json` file defines your Firebase project configuration:

<CodeGroup>
```json firebase.json
{
  "emulators": {
    "auth": {
      "host": "127.0.0.1",
      "port": 9099

    "firestore": {
      "host": "127.0.0.1",
      "port": 8080

      "host": "127.0.0.1",
      "port": 5000

      "port": 4000

    "hub": {
      "host": "127.0.0.1",
      "port": 4400

  },

    "database": "(default)",

    "rules": "firestore.rules",

      "**/node_modules/**"

  },

    "rules": "storage.rules"
  }
}
```
</CodeGroup>

## Firestore Security Rules

Configure security rules in `firestore.rules`:

<CodeGroup>
```javascript firestore.rules
rules_version = '2';

service cloud.firestore {
  match /databases/{database}/documents {
    // Development rules (INSECURE - For local development only)
    match /{document=**} {
      allow read, write: if true;
    }
    
    // Production rules (Recommended)
    // match /{document=**} {
    //   allow read, write: if request.auth != null;
    // }
  }
}
```
</CodeGroup>

<Warning>
  The development rules above allow unrestricted access. Replace with proper authentication rules before deploying to production.
</Warning>

## Firebase Emulators for Local Development

Firebase Emulators allow you to test your application locally without affecting production data.

### Install Firebase CLI

```bash
npm install -g firebase-tools
```

### Initialize Firebase in Your Project

```bash
firebase login
firebase init
```

Select the following services:
- Authentication
- Firestore
- Hosting
- Emulators

### Start the Emulators

```bash
firebase emulators:start
```

The emulators will start on the following ports:

| Service | Port | URL |
|---------|------|-----|
| Authentication | 9099 | http://localhost:9099 |
| Firestore | 8080 | http://localhost:8080 |
| Hosting | 5000 | http://localhost:5000 |
| Emulator UI | 4000 | http://localhost:4000 |
| Emulator Hub | 4400 | http://localhost:4400 |

### Automatic Emulator Connection

TextileFlow automatically detects when running on localhost and connects to emulators:

<CodeGroup>
```javascript Automatic Emulator Detection
if (location.hostname === 'localhost' || location.hostname === '127.0.0.1') {
  console.log("🔧 Usando Firebase Emulators");
  
  try {
    connectAuthEmulator(auth, 'http://localhost:9099', { disableWarnings: true });
    console.log("✓ Auth Emulator conectado");
  } catch (e) {
    console.warn("Auth Emulator ya está conectado o error:", e);
  }
  
  try {
    connectFirestoreEmulator(db, 'localhost', 8080);
    console.log("✓ Firestore Emulator conectado");
  } catch (e) {
    console.warn("Firestore Emulator ya está conectado o error:", e);
  }
} else {
  console.log("✓ Usando Firebase Cloud");
}
```
</CodeGroup>

## Verify Your Setup

After configuration, verify your Firebase connection:

1. Open the browser console (F12)
2. Navigate to your application
3. Look for these messages:
   - "Firebase inicializado correctamente!"
   - "✓ Auth Emulator conectado" (if using emulators)
   - "✓ Firestore Emulator conectado" (if using emulators)

<Note>
  If you see connection errors, double-check your Firebase configuration and ensure all services are enabled in the Firebase Console.
</Note>
