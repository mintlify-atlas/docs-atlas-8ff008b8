## Overview

The Payroll (Pagos) module handles complete salary management for TextileFlow ERP, including base salaries, bonuses, deductions, payment history tracking, and electronic pay slip generation.

<Note>
  The payroll system integrates with the RRHH module for employee data and uses period-based tracking (YYYY-MM) for monthly payments.
</Note>

## Module Architecture

### Core Files

- **JavaScript**: `public/js/pagos.js` (896 lines)
- **HTML**: `public/pages/admin/pagos.html`
- **Database Collections**: `pagos_empleados`, `usuario`, `usuario_admin`

### Payment Flow

```
1. Admin selects period (YYYY-MM)
2. Employee dropdown filters to unpaid employees
3. Salary auto-loads from usuario_admin
4. Admin adds bonuses/deductions
5. System calculates total
6. Payment registered in pagos_empleados
7. PDF pay slip generated on demand
```

async function obtenerTotalPlanillaActiva() {
  const usuariosActivosSnap = await getDocs(
    query(collection(db, "usuario"), where("estado", "==", "activo"))
  );
  const usuariosActivosIds = new Set();
  usuariosActivosSnap.forEach(doc => usuariosActivosIds.add(doc.id));

  const adminSnap = await getDocs(collection(db, "usuario_admin"));
  let total = 0;
  adminSnap.forEach(doc => {

<Steps>
  <Step title="Open Void Modal">

  </Step>
  
  <Step title="Confirm Void">
    System updates payment document (`pagos.js:672-688`):
    ```javascript
    await setDoc(doc(db, "pagos_empleados", id), {
      estado: "anulado",
      motivo_anulacion: motivo,
      fecha_anulacion: new Date()
    }, { merge: true });
    ```
  </Step>
  
  <Step title="Visual Update">
    - Row turns gray with "ANULADO" badge
    - Employee can be paid again for same period
    - Amount removed from totals
  </Step>
</Steps>

<Warning>
  Voiding a payment does NOT delete the record. It marks it as `estado: "anulado"` for audit trail. The employee can be paid again for the same period.
</Warning>

## Helper Functions

### Month Formatting

```javascript
function formatMonthName(mesYao) {
  const [year, month] = mesYao.split('-').map(Number);
  const meses = ["Enero", "Febrero", "Marzo", ...];
  return `${meses[month - 1]} ${year}`;
}
```

### Currency Formatting

```javascript
function formatearSoles(valor) {
  return new Intl.NumberFormat("es-PE", {
    style: "currency",
    currency: "PEN"
  }).format(valor);
}
// Output: "S/ 1,500.00"
```

### Animated Card Values

```javascript
function animateValue(el, start, end, duration = 550) {
  let startTS = null;
  const step = timestamp => {
    if (!startTS) startTS = timestamp;
    const progress = Math.min((timestamp - startTS) / duration, 1);
    el.textContent = formatearSoles(
      Math.floor(progress * (end - start) + start)
    );
    if (progress < 1) requestAnimationFrame(step);
  };
  requestAnimationFrame(step);
}
```

## Common Workflows

### Processing Monthly Payroll

<Steps>
  <Step title="Select Current Month">
    Change month filter to current period (e.g., "Marzo 2026")
  </Step>
  
  <Step title="Verify Active Employees">
    Check "Salario Pendiente" card to see how many employees need payment
  </Step>
  
  <Step title="Process Each Employee">
    Click **Agregar Pago** for each employee:
    - Select period (should be current month)
    - Select employee
    - Verify auto-loaded salary
    - Add bonuses/deductions if applicable
    - Add notes
    - Save
  </Step>
  
  <Step title="Verify Completion">
    When all employees are paid, "Salario Pendiente" should be S/. 0.00
  </Step>
</Steps>

### Correcting Payment Errors

<Steps>
  <Step title="Void Incorrect Payment">
    Find the payment → Click void button → Enter reason (e.g., "Error en monto")
  </Step>
  
  <Step title="Re-register Correctly">
    Employee appears again in dropdown → Register with correct amounts
  </Step>
  
  <Step title="Optional: Hide Voided">
    Toggle "Mostrar anulados" off to hide the voided payment from view
  </Step>
</Steps>
