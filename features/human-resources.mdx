## Overview

The Human Resources (RRHH) module provides complete user lifecycle management for the TextileFlow ERP system, handling employees, clients, and administrators with robust validation and real-time data synchronization.

<Note>
  This module integrates with Firebase Authentication and Firestore for secure user management and real-time updates.
</Note>

## User Management Workflow

<Steps>
  <Step title="Access User Management">
    Navigate to **RRHH** from the admin sidebar. The module displays three tabs:
    - **Usuarios**: All employees and administrators
    - **Clientes**: Customer accounts
    - **Empleados**: Payroll and employee details
  </Step>
  
  <Step title="Create New User">
    Click **Agregar Usuario** button to open the user creation modal.
    
    Required fields:
    - Document Type (CC, CE, DNI, PA, RUC)
    - Document Number (validated by type)
    - First Name
    - Last Name
    - Email (must be unique)
    - Role (Empleado/Administrador/Cliente)
    - Password (min 6 characters)
    
    Optional fields:
    - Phone Number
    - Address
  </Step>
  
  <Step title="Document Validation">
    The system validates document numbers based on type:
    - **DNI**: 8 digits
    - **RUC**: 11 digits
    - **CC**: 6-10 digits
    - **CE**: 9 digits
    - **PA**: 7-9 alphanumeric
    
    Duplicate document validation prevents registration errors.
  </Step>
  
  <Step title="User Activation">
    New users are created with `estado: "activo"` and registered in Firebase Authentication.
    The system stores:
    - User credentials in Auth
    - Profile data in `usuario` collection
    - Timestamp: `fecha_ingreso: serverTimestamp()`
  </Step>
</Steps>

## Employee Payroll Management

### Setting Up Employee Salaries

Employee salary information is stored separately in the `usuario_admin` collection:

```javascript
// From rrhh.js:405
await setDoc(doc(db, "usuario_admin", uid), {
  uid,
  salario: Number,
  fechaIngreso: "YYYY-MM-DD",
  horas_trabajadas: 0,
  bonificaciones: 0,
  descuentos: 0
});
```

### Planilla View (Employee Roster)

The **Empleados** tab displays comprehensive payroll information:

| Column | Source | Description |
|--------|--------|-------------|
| Nombre Completo | `usuario.nombre + apellido` | Employee full name |
| Teléfono | `usuario.telefono` | Contact number |
| Email | `usuario.email` | Email address |
| Fecha de Ingreso | `usuario_admin.fechaIngreso` | Hire date |
| Antigüedad | Calculated | Years/months/days in service |
| Salario | `usuario_admin.salario` | Monthly salary (S/. format) |
| Estado | `usuario.estado` | Active/Inactive |
| Fecha de Salida | `usuario_admin.fechaSalida` | Exit date (if inactive) |

<Warning>
  When inactivating an employee, the system automatically records the exit date (`fechaSalida`) in the `usuario_admin` collection.
</Warning>

## Advanced Features

### Multi-Filter Search

**Usuarios Tab Filters** (`rrhh.js:88-145`):

```javascript
// Live filter implementation
const filteredUsers = allUsers.filter(cliente => {
  // Skip clients in employee view
  if (cliente.rol === 'Cliente') return false;
  
  const matchesSearch = !searchTerm ||
    (cliente.nombre && cliente.nombre.toLowerCase().includes(searchTerm)) ||
    (cliente.apellido && cliente.apellido.toLowerCase().includes(searchTerm)) ||
    (cliente.email && cliente.email.toLowerCase().includes(searchTerm)) ||
    (cliente.documento && cliente.documento.includes(searchTerm));

  const matchesTipoDoc = !filterTipoDoc || cliente.tipo_documento === filterTipoDoc;
  const matchesRol = !filterRol || cliente.rol === filterRol;
  const matchesEstado = !filterEstado || cliente.estado === filterEstado;

  return matchesSearch && matchesTipoDoc && matchesRol && matchesEstado;
});
```

### Status Management

Toggling user status (`rrhh.js:326-393`):

<Tabs>
  <Tab title="Activation">
    When activating a user:
    - Sets `estado: "activo"`
    - Clears `fechaSalida` in `usuario_admin`
    - User regains system access
  </Tab>
  
  <Tab title="Deactivation">
    When deactivating a user:
    - Sets `estado: "inactivo"`

</Tabs>

<Steps>
  <Step title="Create Client Account">
    From **Clientes** tab, click **Agregar Cliente** (role auto-set to "Cliente")

  <Step title="Client Filtering">

</Steps>

## Best Practices

<Note>
  **Salary Management**: Always set employee salary data immediately after user creation to ensure accurate payroll processing.
</Note>

<Warning>
  **Email Uniqueness**: Email addresses must be unique across all roles. The system prevents duplicate emails even if the user was previously deleted from Auth.
</Warning>

<Tip>
  Use the **Limpiar** (Clear Filters) button to reset all search criteria and view the complete user list.
</Tip>
