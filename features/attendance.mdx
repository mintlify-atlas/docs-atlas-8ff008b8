## Overview

The Attendance (Asistencias) module provides comprehensive employee time tracking with clock-in/clock-out functionality, customizable work schedules, tardiness detection, and detailed reporting.

<Note>
  The system uses Lima, Peru timezone (America/Lima) for all attendance records to ensure consistent time tracking.
</Note>

## Attendance Recording

### Employee Self Check-In

Employees mark their attendance from their dashboard:

```javascript
// From asistencias.js:240-271
async function marcarAsistencia(tipo) {
  const hoy = new Date().toISOString().split('T')[0];
  const hora = new Date().toTimeString().slice(0, 5);

  const docId = `${currentUser.uid}_${hoy}_${tipo}`;

  await setDoc(doc(db, "asistencias", docId), {
    userId: currentUser.uid,
    displayName: currentUser.displayName,
    fecha: hoy,  // YYYY-MM-DD
    hora: hora,  // HH:MM
    tipo: tipo,  // "entrada" or "salida"
    timestamp: Timestamp.fromDate(new Date()),
    source: "self",
    createdAt: serverTimestamp()
  });
}
```

<Steps>
  <Step title="Access Dashboard">
    Employee logs in and sees attendance widget with current status
  </Step>
  
  <Step title="Mark Entry">
    Click **Marcar Entrada** button
    - System records current time in Lima timezone
    - Button becomes disabled
    - Exit button becomes enabled
  </Step>
  
  <Step title="Mark Exit">
    Click **Marcar Salida** button when leaving
    - System records exit time
    - Both buttons become disabled
    - Daily record is complete
  </Step>
</Steps>

## Schedule Configuration

### Default Schedule

Stored in `configuracion/horario_default` collection:

```javascript
let defaultSchedule = {
  horaEntrada: "08:00",
  horaSalida: "17:00",
  tolerancia: 15  // minutes
};
```

### Custom Schedules

Stored in `configuracion/horarios_personalizados` collection:

```javascript
customSchedules = {
  "lunes": { horaEntrada: "09:00", horaSalida: "18:00", tolerancia: 10 },
  "viernes": { horaEntrada: "08:00", horaSalida: "16:00", tolerancia: 15 }
};
```

### Schedule Priority

The system applies schedules in this order (`asistencias.js:113-126`):

1. **Employee-specific schedule** (if `employeeSchedules[userId].activo`)
2. **Day-of-week custom schedule** (e.g., "lunes", "martes")
3. **Default schedule** (fallback)

## Tardiness Detection

### Automatic Late Marking

The system calculates tardiness when displaying attendance (`asistencias.js:410-424`):

```javascript
if (emp.horaEntrada) {
  const [horaInicioH, horaInicioM] = horaInicio.split(':').map(Number);
  const minutosInicio = horaInicioH * 60 + horaInicioM + tolerancia;

  const [h, m] = emp.horaEntrada.split(':').map(Number);
  const minutosEntrada = h * 60 + m;

  if (minutosEntrada > minutosInicio) {
    esTarde = true;
    tardanzas++;
    statusClass = "late";
  }
}
```

**Example**:
- Default entry time: 08:00
- Tolerance: 15 minutes
- Grace period until: 08:15
- Entry at 08:20 = **Late** (5 minutes over tolerance)

## Today's Attendance View (Admin)

### Real-Time Dashboard

Administrators see live attendance data (`asistencias.js:321-451`):

| Column | Description | Calculation |
|--------|-------------|-------------|
| Nombre | Employee name | From `usuario` collection |
| Hora Inicio | Scheduled entry | From applicable schedule |
| Entrada | Actual entry time | From attendance record |
| Hora Fin | Scheduled exit | From applicable schedule |
| Salida | Actual exit time | From attendance record |
| Horas | Hours worked | `(exitTime - entryTime) / 60` |
| Estado | Status badge | Completo/Incompleto/Tarde/Ausente |
| Acciones | Edit button | Opens edit modal |

### Status Badges

<Tabs>
  <Tab title="Completo">
    **Green badge**: Employee has entry and exit recorded
    - Both timestamps present
    - Hours worked calculated
    - Counts toward "presentes"
  </Tab>
  
  <Tab title="Incompleto">
    **Yellow badge**: Entry recorded but no exit
    - Missing exit timestamp
    - Still counted as present
    - Hours shows "--"
  </Tab>
  
  <Tab title="Tarde">
    **Orange badge**: Arrived after grace period
    - Entry time > (scheduled + tolerance)
    - Increments tardiness counter
    - Overrides other statuses
  </Tab>
  
  <Tab title="Ausente">
    **Red badge**: No entry recorded
    - Employee not in attendance list
    - Calculated as: total employees - present
  </Tab>
</Tabs>

## AttendanceService API

### Service Methods

The `attendanceService.js` provides structured API:

```javascript
// Mark attendance (self-registration)
await attendanceService.markAttendance(
  userId, userEmail, displayName, role, type, notes
);

// Register manual attendance (admin)
await attendanceService.registerManualAttendance(
  adminId, employeeData, date, time, type, notes
);

// Get today's attendances
const result = await attendanceService.getTodayAttendances();

// Get attendance by date
const result = await attendanceService.getAttendancesByDate("2026-03-08");

// Get user attendances in range
const result = await attendanceService.getUserAttendances(
  userId, startDate, endDate
);

// Update attendance
await attendanceService.updateAttendance(
  attendanceId, updates, adminId
);
```

interface ServiceResponse {
