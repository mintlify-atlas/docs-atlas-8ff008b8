## Overview

The Inventory module provides comprehensive product and stock management for TextileFlow ERP, with support for product variants (attributes and values), category organization, and real-time stock tracking.

<Note>
  The inventory system uses Firebase Firestore with optimized parallel queries for fast performance even with large product catalogs.
</Note>

## Module Architecture

### Core Files

- **JavaScript**: `public/js/inventario.js` (907 lines)
- **HTML**: `public/pages/admin/inventario.html` (236 lines)
- **Database Collections**: `articulos`, `categoria`, `atributo`, `valor`, `stock_inventario`

### Data Model

The inventory system uses a normalized database structure:

```
articulos (products)
├── categoria (categories)
├── atributo (attributes: size, color, etc.)
│   └── valor (attribute values: S, M, L, etc.)
└── stock_inventario (stock per variant)
```

## Product Management

### Creating a Product

<Steps>
  <Step title="Open Product Modal">
    Click **Nuevo Producto** button to open the product creation form
  </Step>
  
  <Step title="Enter Product Details">
    Required fields:

    - **Precio base**: Base price (S/.)

    - **Descripción**: Product description
    - **Imagen**: Product photo (currently disabled)
  </Step>
  
  <Step title="Product Registration">
    The system creates two documents:
    
    **Articulo Document** (`inventario.js:602-622`):
    ```javascript
    {
      nombre: string,
      descripcion: string | null,
      precio_base: number,
      id_categoria: string,  // doc reference ID
      estado: 'activo',
      fecha_registro: serverTimestamp()
    }
    ```
    
    **Stock Document** (`inventario.js:624-635`):
    ```javascript
    {
      id_categoria: string,
      id_stock: string,  // auto-generated
      id_articulo: string,  // links to articulo
      id_atributo: number,  // attribute correlative
      id_valor: number,  // value correlative
      stock: number,
      fecha_registro: serverTimestamp()
    }
    ```
  </Step>
  
  <Step title="View in Catalog">
    Product appears in main table with aggregated stock from all variants
  </Step>
</Steps>

### Product Table Display

The product table (`inventario.js:392-512`) shows real-time data with optimized queries:

| Column | Source | Description |
|--------|--------|-------------|
| Imagen | - | Placeholder (image upload disabled) |
| Nombre | `articulos.nombre` | Product name |
| Categoría | `categoria.nombre` | Category via join |
| Característica | `atributo.nombre_atributo` | Attribute names (e.g., "Talla, Color") |
| Detalle | `valor.valor` | Attribute values (e.g., "M, Rojo") |
| Precio Base | `articulos.precio_base` | Base selling price |
| Stock | Sum of `stock_inventario.stock` | Total inventory |
| Descripción | `articulos.descripcion` | Product notes |
| Estado | `articulos.estado` | Active/Discontinued |
| Acciones | - | Edit and status buttons |

<Warning>
  Stock is automatically calculated by summing all variant quantities. The system performs parallel queries for optimal performance.
</Warning>

## Stock Management

### Stock Tracking Model

Each product variant has a separate stock record:

```typescript
interface StockInventario {
  id_categoria: string;

  id_valor: number;     // e.g., "M"
  stock: number;
  fecha_registro: Timestamp;
  fecha_actualizacion?: Timestamp;
}
```

## Performance Optimization

### Parallel Query Strategy

The module uses Promise.all() for parallel database queries (`inventario.js:401-443`):

```javascript
// 1. Fetch all articles
const articulosProcesados = snapshot.docs.map(async (artDoc) => {
  const art = { id: artDoc.id, ...artDoc.data() };

  // 2. Parallel queries for this article
  const [categoriaSnap, stockSnap] = await Promise.all([
    getDoc(doc(db, "categoria", art.id_categoria)),
    getDocs(query(collection(db, "stock_inventario"), 
                  where("id_articulo", "==", art.id)))
  ]);

  // 3. Parallel queries for stock attributes
  let stockPromises = [];
  for (const sDoc of stockSnap.docs) {
    const attrValPromise = Promise.all([
      getDocs(query(collection(db, "atributo"), ...)),
      getDocs(query(collection(db, "valor"), ...))
    ]);
    stockPromises.push(attrValPromise);
  }

  const resultadosStocks = await Promise.all(stockPromises);
  return { art, categoriaNombre, stockTotal, atributosNombres, valoresNombres };
});

// Wait for all articles to finish
const resultadosFinales = await Promise.all(articulosProcesados);
```
